## 一、论文动机与问题背景

### 现有方案的局限性

当前主流大型多模态模型（如 Gemini、GPT-4o、Qwen 等）虽然能以自回归方式理解多模态输入，但**输出仍以文本为中心**。为了支持多模态生成（如图像、视频、音频），这些系统通常采用**后融合（Late-Fusion）**架构——在预训练好的语言主干模型上附加模态特定的解码器/生成器。这种设计的缺陷在于：

1. **能力跷跷板问题（Ability Seesaw）**：多模态生成能力的增强往往以牺牲语言核心性能为代价；
2. **模态间割裂**：不同模态使用非自回归目标，阻碍了深层跨模态整合；
3. **扩展性受限**：模态专用的专家分配往往依赖启发式规则，在三个以上模态时极难设计。

### 核心问题

> 如何在一个统一的自回归框架中，**原生**支持文本、图像、视频、音频的理解与生成，同时保持各模态的强能力，并随模型和数据规模的增长有效扩展？

## 二、核心方法与贡献

### 2.1 统一自回归架构（Next-Group-of-Tokens Prediction）

ERNIE 5.0 的核心创新是将所有模态**从零开始联合训练**，不依赖预训练好的语言模型。所有异质输入被投影到**共享 token 空间**，统一在 **Next-Group-of-Tokens Prediction** 目标下优化：


| 模态    | 生成范式                                                                                |
| ----- | ----------------------------------------------------------------------------------- |
| 文本    | Next-Token Prediction (NTP) + Multi-Token Prediction (MTP)                          |
| 图像/视频 | **Next-Frame-and-Scale Prediction (NFSP)**：帧内从低到高分辨率（Next-Scale），帧间时序预测（Next-Frame） |
| 音频    | **Next-Codec Prediction (NCP)**：跨 Transformer 层的层级码字预测                              |


### 2.2 超稀疏 MoE + 模态无关专家路由

为了在控制计算开销的同时实现万亿级参数容量，ERNIE 5.0 采用**超稀疏 Mixture-of-Experts（MoE）架构**，激活率低于 **3%**。

关键设计——**模态无关专家路由（Modality-Agnostic Expert Routing）**：

- 路由决策基于**统一 token 表示**，而非显式的模态标识符；
- 文本、图像、视频、音频 token 均被分发到**共享的专家池**；
- 无需启发式设计模态专用专家，专家专业化通过自监督学习自然涌现；
- 使用**无辅助损失的负载均衡**（Auxiliary-Loss-Free Load Balancing）确保万亿参数规模下的稳定专家利用率。

**路由行为可视化发现**：

- 部分专家被所有模态频繁激活（通用专家），其余专家呈现强模态特异性；
- 从浅层到深层，跨模态专家重叠增加，表明深层表示向高层统一语义聚合；
- 将推理时 top-k 减少至 25%，解码速度提升 **>15%**，精度仅有轻微损失。

### 2.3 弹性训练范式（Elastic Training）

**传统方案的痛点**：训练后压缩（剪枝/蒸馏）需要额外的专用流程，且每换一个尺寸就要重来一次。

**弹性训练**：在**单次预训练运行**中，同时优化一族不同容量的子网络，沿三个正交维度引入结构灵活性：


| 弹性维度                        | 机制                        | 训练概率分配                |
| --------------------------- | ------------------------- | --------------------- |
| **弹性深度（Elastic Depth）**     | 随机变化激活的 Transformer 层数    | 全深度 75%，子网络 25%       |
| **弹性宽度（Elastic Width）**     | 随机变化 MoE 层的专家总数           | 全宽度 80%，缩减宽度 20%      |
| **弹性稀疏度（Elastic Sparsity）** | 随机变化每 token 激活的专家数（top-k） | 默认 k 值 80%，缩减 k 值 20% |


**关键实验结果**：

- 仅使用 **35.8% 的总参数** 和 **53.7% 的激活参数**的弹性变体，平均分（75.17）与全模型（75.55）相当；
- 弹性深度还带来轻微的正则化效果，全深度验证损失略有改善。

### 2.4 视觉建模

**视觉 Tokenizer**：

- 使用因果 2D 多尺度 Tokenizer（图像）→ 膨胀为因果 3D 卷积 Tokenizer（统一图像和视频）；
- 位量化（Bit-wise Quantization）策略 + **渐进式 Tokenizer 切换**（从低 bit 到高 bit），平滑优化轨迹；
- 辅助对抗损失（GAN discriminator）+ 语义正则化损失提升表示质量。

**双路混合视觉表示（Dual-Path Hybrid Representation）**：

- 融合 CNN 感知特征（局部细节）和 ViT 语义特征（高层语义）；
- 使用**基于注意力的 Patch Merger**代替简单 MLP 融合，避免表示干扰；
- 图像分组 K=4 相邻 patch，视频分组 K=16（跨 4 帧）。

**统一时空位置编码（Uni-RoPE）**：为文本、视觉、音频 token 提供统一的 (t, h, w) 三元组位置编码，并采用中心对齐坐标策略保证多尺度空间一致性。

**级联扩散精化器**：自回归主干生成低分辨率语义结构，独立训练的扩散模型负责高分辨率细节增强，避免优化冲突。

### 2.5 音频建模

**音频 Tokenizer**：

- 12.5 Hz 的 codec 式 tokenizer + RVQ（残差向量量化）；
- 第一层 token 语义信息（蒸馏自 Whisper 模型），残差层保留音色、韵律等声学细节。

**Next-Codec Prediction (NCP)**：

- 在 Transformer 顶层插入多个音频头，实现**层级深度自回归预测**；
- 每层处理一个粒度级别，避免将所有 RVQ token 展平为超长序列；
- 生成阶段逐层生成码字，每层预测结果反馈到隐状态以条件化下层预测。

### 2.6 后训练：统一多模态强化学习

两阶段后训练：SFT → **统一多模态强化学习（UMRL）**

针对超稀疏 MoE + 多模态 RL 训练的三大挑战，提出对应解决方案：

**① 无偏重放缓冲区（U-RB）**：

- 问题：长尾 Rollout 响应导致 GPU 空闲，直接截断（APRIL 方案）引入数据难度分布偏移；
- 解决：引入数据排序约束，仅允许当前迭代分配的数据组参与后续训练，保持训练数据分布无偏。

**② 多粒度重要性采样裁剪（MISC）+ 已掌握正样本掩码（WPSM）**：

- 问题：序列级截断重要性采样会剪掉大量低熵响应，导致训练-推理数值不匹配，引发熵坍塌；
- MISC：将 token 级掩码（IcePop）与序列级重要性比（GSPO）结合，平衡探索-利用；
- WPSM：追踪每个查询的成功率，屏蔽已熟练掌握的样本，将梯度预算导向困难样本。

**③ 自适应提示强化学习（AHRL）**：

- 问题：困难任务全零奖励，无有效梯度信号；
- 解决：向查询注入部分 "思路骨架"（前 p_hint 比例的思考 token），随训练进展按退火计划逐渐减少提示比例，直至完全自主探索。

## 三、训练流程

### 预训练数据

- **文本**：多语言网页、书籍、代码、科学文献（UTF-16BE 编码 + BPE Dropout）；
- **多模态**：图文对、视频文本对、音频文本对及交错多模态序列；
- 总规模：数万亿 text token + 大量多模态样本。

### 训练 Recipe（两阶段）

1. **Stage 1（8K 上下文）**：Warmup-Stable-Decay 学习率调度，批次大小从 14M → 56M tokens，RoPE base = 1,000,000；
2. **Stage 2（32K + 128K 中训练）**：余弦学习率 1e-4 → 1e-5，引入后验损失加权策略平衡跨模态训练信号。

### 基础设施（PaddlePaddle）

- **混合并行**：4路张量并行 + 12路流水线并行 + 64路专家并行 + ZeRO-1 数据并行 + 上下文并行（长文本）；
- **DeepEP** 高效跨节点通信；
- **FP8 混合精度训练** + 动态自适应激活值 Offloading；
- **FlashMask**：支持跨样本异构注意力掩码，相比 FlexAttention 在算子级别提速 200%，端到端训练加速 >20%；
- **分解式 RL 基础设施**：训练引擎、推理引擎、环境交互异步协调；统一 FP8 执行引擎消除数值不一致；弹性 CPU 池化利用空闲 CPU 资源驱动大规模环境 Rollout。

## 五、结论与未来方向

**主要结论**：

1. ERNIE 5.0 是**首个公开披露的、支持多模态理解与生成的万亿参数统一自回归模型**；
2. 模态无关专家路由证明了在不引入模态专用设计的情况下，专家专业化可自然涌现；
3. 弹性训练是一种**原则性训练范式**（而非事后压缩），单次预训练即可派生多个尺寸的可部署子模型；
4. 统一自回归预训练为多模态能力提供了坚实基础，后训练主要增强指令遵循和推理，而非补偿感知缺陷。

**未来方向**：

- 表示维度弹性（Elastic Hidden Size）；
- 更强的长链路复杂推理能力（对标 Gemini 3-Pro 的差距）；
- 层感知专家分配、自适应均衡策略等下一代原生多模态架构原则。

### Q1：帮我详细介绍统一自回归范式，各模态输入统一模型前的表达形式是什么？

**A：**

ERNIE 5.0 的"统一自回归范式"核心是：无论输入是文本、图像、视频还是音频，都要在进入统一主干（Backbone）之前，被转换成**形状为 `ℝ^{D_backbone}` 的连续浮点向量序列**，与文本 token embedding 形式完全对齐，从而可以拼接成一条统一的序列送入 Transformer。

但不同模态到达这一形式的**路径差异很大**：

---

#### 文本

最简单，走标准 NLP 路径：

```
原始文本
  ↓ BPE 分词（UTF-16BE 编码，启用 BPE Dropout）
  ↓ token id（整数）
  ↓ Embedding Table 查表
  → ℝ^{D_backbone}  （一个 token = 一个向量）
```

位置编码使用 Uni-RoPE，文本的 `(t, h, w)` 三元组退化为序列位置的一维编码。

---

#### 视觉（图像 / 视频）

视觉模态根据任务目标分为**两条完全不同的路径**（图像被当作单帧视频统一处理）：

**路径 1：理解路径（连续特征，不量化）**

目的是最大程度保留视觉语义细节，不做量化。

```
图像/视频帧
  ├── CNN 分支 → F_cnn ∈ ℝ^{N×K×D_cnn}   （局部感知特征，含细节纹理）
  └── ViT 分支 → F_vit ∈ ℝ^{N×K×D_vit}   （高层语义特征，含全局语义）
       │
       ├── CNN 特征投影对齐到 D_vit 维度
       ├── 拼接 → F_mrg ∈ ℝ^{N×2K×D_vit}
       ├── 多头自注意力（Attention-Based Patch Merger）
       ├── mean pooling over patch 维度 → F_out ∈ ℝ^{N×D_vit}
       └── 线性投影 → ℝ^{D_backbone}  （直接送入主干，全程连续）

图像理解：K=4（2×2 相邻 patch 分为一组）
视频理解：K=16（4 帧 × 每帧 4 个 patch）
```

这里用「Attention-Based Patch Merger」而非简单 MLP 融合，是为了避免 CNN 和 ViT 两类异质特征互相干扰（MLP 会把两者强行混合，自注意力能自适应地决定用哪个分支的信息）。

**路径 2：生成路径（量化为离散 id，再查表）**

自回归生成必须预测「下一个 token」，因此必须先把连续特征离散化为整数 id。

```
图像/视频 → 因果 3D 卷积 Tokenizer（统一图像和视频）→ 连续 latent
  ↓ Bit-wise 量化
    （对 latent 的每一位二进制独立量化，b 位拼成一个整数 id，词表大小 = 2^b）
  ↓ Visual Embedding Table 查表（与文本 embedding table 结构相同）
  → ℝ^{D_backbone}

• 训练策略：渐进式 Tokenizer 切换，从低 bit（粗粒度，词表小）逐步切换到高 bit（细粒度，词表大），平滑优化轨迹
• 多尺度：NFSP 中每个 scale 独立量化，序列排列为 Scale_1 → Scale_2 → Scale_3
```

**Bit-wise 量化 vs VQ-VAE**：VQ-VAE 是把整个向量 nearest-neighbor 映射到 codebook 条目（词表大小固定，与向量维度耦合）；Bit-wise 量化是对每个 bit 独立二值化，词表大小 = 2^b，可以通过改变 bit 数灵活控制粒度，天然支持渐进式切换。

**Uni-RoPE 位置编码**：视觉 token 携带 `(t_i, h_i, w_i)` 三元组——`t` 为帧索引，`(h, w)` 为帧内行列坐标；多尺度间采用中心对齐坐标策略，保证相同空间位置在不同分辨率 scale 下坐标一致。

**NFSP 注意力掩码**：同一 scale 内的 token 彼此**双向可见**（并行预测，加速训练），不同 scale 之间严格**因果单向**（低 scale → 高 scale，确保条件依赖方向正确）。

---

#### 音频

**理解时（加法叠加，序列长度不随 RVQ 层数 L 爆炸）**：

```
音频波形
  ↓ RVQ codec（12.5 Hz 帧率，L 层残差向量量化）
  每个时间步产生 L 个 code id：[c_1, c_2, ..., c_L]
    Level 1：高层语义 code（由 Whisper encoder 蒸馏训练，理解语音内容）
    Level 2~L：残差声学 code（保留音色、韵律、细节）

  token_embedding_t = Σ_{l=1}^{L} E_l[c_l]
    （L 个独立 embedding 矩阵各自查表后加法叠加）
  → ℝ^{D_backbone}，序列长度 = 时间步数（与 L 无关，不爆序列）
```

若直接把 L 层 code 展平，序列长度 = 时间步 × L，长度爆炸；加法叠加将 L 层信息压缩进同一维度，兼顾多粒度信息又控制序列长度。

**生成时（NCP，跨 Transformer 层深度自回归预测 RVQ 层级）**：

```
Transformer 主干的顶层插入 L 个 Audio Head（每个 Head 对应一层 RVQ）：
  第 1 个 Head → 预测 c_1（最高层语义 code）
    → 将 E_1[c_1] 加回当前隐状态 h，条件化后续预测
  第 2 个 Head → 预测 c_2（条件于 c_1 和更新后的 h）
    → 将 E_2[c_2] 加回 h
  ...
  第 L 个 Head → 预测 c_L
  全部 L 个 code → RVQ 解码器 → 重建音频波形

  语音合成时额外注入 Speaker Embedding，控制目标说话人音色
```

---

#### 汇总对比


| 模态     | 进入主干前的形式                                         | 路径特点        |
| ------ | ------------------------------------------------ | ----------- |
| 文本     | token id → embedding 查表 → 向量                     | 离散 id，最简单   |
| 视觉（理解） | CNN+ViT 双路特征 → Patch Merger → 投影 → 向量            | 全程连续，不量化    |
| 视觉（生成） | 连续 latent → Bit-wise 量化 → id → embedding 查表 → 向量 | 必须离散化才能自回归  |
| 音频（理解） | RVQ 各层 code → 各层 embedding 加法叠加 → 向量             | 多层信息压缩入同一向量 |
| 音频（生成） | 与理解相同的输入侧；输出侧跨层深度自回归预测各层 code                    | NCP 机制      |


所有路径的最终结果形式统一：**长度可变的 `ℝ^{D_backbone}` 向量序列**，拼接后进入同一个 Transformer 主干，主干本身无需感知 token 来自哪个模态。

---

### Q2：视觉模态最终产生的 token 是什么形式的？维度多少？

**A：**

视觉 token 和文本 token 进入主干前形式完全相同：**一个 token = 一个 `D_backbone` 维的浮点向量**，维度与文本 embedding 相同（论文未公开具体数值，记为 `D_backbone`）。

但不同任务下得到这个向量的方式不同：

- **理解任务**：经过 CNN+ViT 双路 → Patch Merger → mean pooling，得到 `F_out ∈ ℝ^{N×D_vit}`，再线性投影到 `ℝ^{D_backbone}`。每 K 个 patch 压缩成 1 个 token（图像 K=4，视频 K=16），所以序列长度 = 原始 patch 数 / K。
- **生成任务**：经过 Bit-wise 量化后得到离散整数 id，查 Visual Embedding Table 得到 `D_backbone` 维向量。序列长度 = 各尺度量化后的空间 token 数之和（多尺度 NFSP）。

---

### Q3：视觉模态的 embedding 是如何转成 token 的？（理解路径的 embedding 是连续向量，怎么做自回归预测？）

**A：**

这里需要区分**理解**和**生成**两种任务：

**理解任务不需要预测「下一个视觉 token」**，只需要理解视觉内容。模型接收视觉连续向量作为输入（就像接收文本 embedding 一样），然后用文本 token 来输出回答。整个过程中视觉特征始终是连续向量，不需要量化，也不需要从视觉词表采样。

**生成任务必须输出离散 id**，因为自回归预测的目标是「预测下一个 token 的概率分布」，概率分布需要定义在有限词表上，而连续向量无法直接做这件事。所以：

1. 训练目标侧：把真实图像的视觉 latent 做 Bit-wise 量化，得到离散 id 作为预测目标
2. 模型输出侧：最后一层接视觉 token 分类头，输出在 `2^b` 大小词表上的 softmax 分布
3. 解码时：从分布中采样 id → 查 Embedding Table → 作为下一步的输入
4. 所有 id 生成完毕后 → 反量化 → 视觉解码器 → 最终图像/视频

**核心区分**：

- 理解：视觉 embedding → 直接当输入向量，模型输出文本回答
- 生成：视觉 latent → 量化成 id → 自回归预测 id → 反量化 → 重建图像

两条路在进入主干后的向量形式相同，但训练目标和解码逻辑完全不同。

### Q4：训练时用更多专家、推理时用少一点，这个思路好像挺容易的，之前有类似的工作吗？

**A：**

这个方向确实已有先驱工作，但 ERNIE 5.0 的做法有重要区别：

**相关先驱工作**：


| 工作                                       | 思路                                     | 局限                               |
| ---------------------------------------- | -------------------------------------- | -------------------------------- |
| **LayerDrop**（Fan et al., 2020）          | 训练时随机 drop 若干层，推理时可以用更浅的子网络            | 只有深度一个维度，且主要用于 fine-tuning       |
| **Once-For-All (OFA)**（Cai et al., 2020） | 在 fine-tuning 阶段同时训练一族不同尺寸的子网络（深度、宽度等） | 仅用于 CNN/小型模型的 fine-tuning，非预训练规模 |
| **Matformer / Flextron**（2023-2024）      | 将 OFA 思路扩展到 Transformer，支持弹性宽度推理       | 仍是 fine-tuning 或监督训练阶段，规模小，非 MoE |
| **Expert Dropout**                       | MoE 训练时随机屏蔽部分专家                        | 目的是改善负载均衡，不是为了推理时用更少专家           |
| **直接减少 top-k 推理**                        | 不做特殊训练，推理时直接把 top-k 从默认值降低             | 存在训练-推理分布偏移，精度损失大                |


**ERNIE 5.0 的新贡献**：

1. **规模**：首次在万亿参数级 MoE **预训练阶段**引入这一思路（OFA 类工作都是 fine-tuning 阶段）
2. **三维度**：同时覆盖深度（层数）+ 宽度（专家总数）+ 稀疏度（top-k），而非单一维度
3. **训练感知**：子网络在预训练中被同步优化，推理降 top-k 时没有分布偏移——这是"容易说，难做"的部分，naive 地直接在推理时降 k 效果很差，必须训练阶段一起配合

所以这个思路「概念上容易理解」，但在万亿参数 MoE 预训练中落地并验证有效是 ERNIE 5.0 的实质贡献。

---

### Q5：top-k 是啥？是激活的专家数量吗？

**A：**

是的，**top-k 就是每个 token 在路由时激活的专家数量**。

MoE 层的完整工作流程：

```
输入 token x ∈ ℝ^D
  ↓ Router（一个小型线性层）计算 logits
  ↓ Softmax → 每个专家的得分 [s_1, s_2, ..., s_E]
  ↓ 取前 k 个最高分 → 激活专家集合 {i_1, ..., i_k}，对应权重 {g_1, ..., g_k}
  ↓ 并行执行这 k 个专家
  ↓ 加权求和：output = Σ_{j=1}^{k} g_j · FFN_{i_j}(x)
```

**几个重要数字的含义**：

- 总专家数 E：模型中有多少个 FFN 专家（决定参数量）
- 激活专家数 k（即 top-k）：每个 token 实际用多少个（决定计算量）
- 激活率 ≈ k/E：ERNIE 5.0 激活率 < 3%（E 非常大，k 相对很小）

所以「总参数量」取决于 E，「激活参数量」（每次前向计算实际用到的）取决于 k，两者可以独立调整，这是 MoE 的核心优势。

---

### Q6：专家输出的结果不是通过拼接吗？

**A：**

不是拼接，是**加权求和**。这是 MoE 和 Multi-Head Attention 的一个容易混淆的区别：

**MoE（加权求和）**：

```
每个专家 FFN_i: ℝ^D → ℝ^D  （输入 D 维，输出也是 D 维）
output = Σ_{i ∈ top-k} g_i · FFN_i(x) ∈ ℝ^D

无论 k 是多少，output 始终是 D 维
```

**Multi-Head Attention（拼接）**：

```
每个 head 的 value 投影: ℝ^D → ℝ^{D/h}  （h 个 head 各自 D/h 维）
各 head 输出拼接: [head_1; head_2; ...; head_h] → ℝ^{D/h × h} = ℝ^D
再做一次线性投影 W_O

h 的数量固定，维度平衡靠 D/h 设计保证
```

**为什么 MoE 要用加权求和而非拼接？**

- 拼接的话，k 个专家输出拼起来是 `k×D` 维，还需要额外的投影层压回 D 维，参数量和计算量随 k 线性增长，丧失了 MoE 的稀疏性优势
- 加权求和保持输出维度固定为 D，k 变化不影响下游层的接口，这是弹性稀疏度（弹性地改变 top-k）能够成立的**数学基础**

**三种"减少专家"方式的对比**：


| 方式             | 减少的是        | 参数量 | 显存  | 计算量 | 输出维度   | ERNIE 5.0 支持 |
| -------------- | ----------- | --- | --- | --- | ------ | ------------ |
| 降 top-k（弹性稀疏度） | 激活专家数       | 不变  | 不变  | ↓   | **不变** | ✅            |
| 降专家总数 E（弹性宽度）  | 专家总数        | ↓   | ↓   | ↓   | **不变** | ✅            |
| 降隐层维度 D（弹性隐层）  | hidden size | ↓↓  | ↓↓  | ↓↓  | **变化** | ❌（未来工作）      |


降 D 之所以难，是因为 hidden size 贯穿所有层的所有矩阵，改变它就要改变全模型所有参数的形状，是"牵一发动全身"的系统工程。

### Q7：ERNIE 5.0-Exp 推理时怎么进行子网络抽取？

**A：**

论文中呈现了**两种不同的推理时弹性使用方式**，需要区分清楚：

---

#### 方式 A：仅应用弹性稀疏度（无需额外训练，零成本）

这是最轻量的方式，只改变推理时的 top-k 参数：

```
✅ 使用完整的 ERNIE 5.0-Exp checkpoint（参数全部加载）
✅ 推理时只修改 top-k 路由参数，其余不变
→ 变体名称：ES₂₅.₀%（Elastic Sparsity，激活 25% 的默认 top-k）

结果：
  - 解码速度提升 >15%（激活的 FFN 计算减少）
  - 平均评测分：74.43 vs 全模型 75.55（差距约 1 分）
  - 显存需求：不变（所有专家权重仍需加载）
```

这种方式解决的是**在线延迟**问题，不解决显存问题。

---

#### 方式 B：三维联合弹性抽取（需要额外训练流程）

这是从大模型中"派生"出一个真正更小的独立模型：

```
Step 1：从 ERNIE 5.0-Exp-Base（预训练 checkpoint）物理截取参数
  - 移除若干 Transformer 层（弹性深度）
  - 只保留部分专家的权重（弹性宽度）
  - 调整 top-k 配置（弹性稀疏度）
  → 得到体积更小的 checkpoint（35.8% 总参数，53.7% 激活参数）

Step 2：以截取后的小模型为起点，做 mid-training
  （继续在数据上训练，让小模型适应自身容量）

Step 3：对小模型做完整 post-training（SFT + UMRL）

→ 变体名称：EA₃₅.₈%（Elastic All，三维联合弹性）

结果：
  - 总参数 35.8%，激活参数 53.7%
  - 平均评测分：75.17 vs 全模型 75.55（仅差 0.38）
  - 显存大幅下降（物理上更少的权重）
```

**论文未披露的工程细节**：具体如何选择截取哪些层、保留哪些专家（均匀截取？重要性排序？贪心搜索？）未在论文中说明，是实现细节。

---

#### 两种方式的直观对比


|      | 方式 A（ES₂₅.₀%）     | 方式 B（EA₃₅.₈%）                   |
| ---- | ----------------- | ------------------------------- |
| 操作   | 改参数，不改 checkpoint | 截取 checkpoint → 重新训练            |
| 额外成本 | 零                 | 需要 mid-training + post-training |
| 显存   | 不变（全量专家仍在）        | 大幅减少（物理上更小的模型）                  |
| 推理延迟 | 降低（少算 FFN）        | 降低（更小模型）                        |
| 性能   | 74.43（差全模型约 1 分）  | 75.17（差全模型约 0.4 分，反而更好）         |
| 适用场景 | 延迟敏感的在线服务         | 显存受限的部署环境                       |

#### 弹性训练为什么比传统压缩更好？

传统做法：训练完整模型 → 剪枝/蒸馏 → 每换一个目标尺寸就重来一次。

弹性训练：**一次预训练**，同时优化全模型和所有子网络配置。子网络权重来自大模型参数的子集，天然继承大模型的表示能力；mid-training 只需微调适应，而非从头学习，效率极高。

这是"原则性训练范式"而非"事后压缩"的关键所在。
